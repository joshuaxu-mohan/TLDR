# Daily Digest

Personal content digest pipeline: ingests newsletters and podcasts, transcribes audio via Groq Whisper API, summarises with Gemini, serves a daily digest web app with Telegram notifications.

## Architecture

- Backend: FastAPI serving both API and built React frontend as a single process
- Database: SQLite at data/digest.db
- Transcription: Groq Whisper API (whisper-large-v3). No local Whisper. Stops gracefully on 429.
- Summarisation: Gemini 3.1 Flash Lite Preview (fallback: gemini-2.5-flash)
- Frontend: React + Tailwind, built to frontend/dist/, served by FastAPI StaticFiles
- All API routes under /api prefix via APIRouter(prefix="/api")
- Episode discovery: Taddy for metadata + audio URLs only (no transcript fetches)
- Podcast scrapers: 9 modules in src/ingestors/scrapers/ — see .claude/rules/scrapers.md for details. Groq fallback on any scraper failure.
- Notifications: Telegram Bot API (one-way, post-pipeline summary to personal chat)
- Deployment: Windows Task Scheduler (current), Oracle Cloud ARM VM planned

## Transcript Priority (two tiers only)

- `always`: auto-transcribed every pipeline run via scraper then Groq fallback
- `on_demand`: ingested with podcast description, user triggers transcription from frontend
- Newsletters get full content from RSS regardless of priority
- Only 8 sources are `always`: Lex Fridman, Dwarkesh, Decoder, Cheeky Pint, Exchanges, Bloomberg Tech, Prof G Markets, Stratechery. All others are `on_demand`.

## Pipeline Flow

RSS + Taddy (metadata only) → Scrapers (always-priority, freshly ingested first) → Early summarise + digest + feed summary → Groq transcription (max 5/run, budget-aware) → Post-Groq summarise + digest + feed summary → Telegram notification → Exit

Feed summary generates two segments: NEWS SUMMARY (news sources) and INFORMATIVE HIGHLIGHTS (informative sources), stored as JSON with both keys.

## Commands

- Run pipeline once: `python -m src.main --run-now --since 2h`
- Run with scheduler: `python -m src.main --daemon`
- Ingest only: `python -m src.main --ingest-only`
- Start server: `uvicorn src.delivery.api:app --host 0.0.0.0 --port 8000`
- Build frontend: `cd frontend && npm run build && cd ..`
- Windows build + serve: `cd C:\Daily\frontend && npm run build && cd C:\Daily && & "C:\Users\xujos\AppData\Local\Python\bin\python.exe" -m uvicorn src.delivery.api:app --host 127.0.0.1 --port 8000`

## Key Conventions

- src/ingestors/scrapers/runner.py routes to scrapers, exposes get_scraper_for_source() for both pipeline and on-demand transcription
- API field `content_type` (news/informative) maps to DB column `content_category` — keep this mapping consistent in both CreateSourceRequest and UpdateSourceRequest
- Source type colours in frontend: orange (#E8813B) for newsletters, green (#3ECF6E) for podcasts, cyan (#00E5FF) for interactive elements
- Deep dive prompts differ by content_category: news uses structured format, informative uses narrative-following format
- Use datetime.now(datetime.UTC), NEVER datetime.utcnow()
- SQLite Row access: bracket notation with `in row.keys()` guard, NEVER .get()
- Lazy imports for circular dependency avoidance (summariser imported inside functions)
- Error handling: try/except per pipeline step, log and continue, never crash the run
- UK English in all user-facing text and Gemini prompts
- Timezone-aware datetimes everywhere: normalise naive datetimes to UTC before comparison

## Groq Budget

- 120 minutes/hour, 480 minutes/day (free tier)
- Pipeline checks budget before each episode, stops if <5 min remaining
- On 429: parses Retry-After, stops batch if wait >60s, carries to next run
- Skips audio under 2MB (trailers/promos)
- All transcriptions logged to transcription_log table for budget tracking

## Gemini Budget

- 500 RPD for Flash Lite Preview, 250 RPD for 2.5 Flash
- Each summarisation call logged to gemini_log table
- GET /api/gemini-budget returns used/remaining/limit for current day
- GeminiBudget component displays alongside GroqBudget, refreshes on gemini-budget-refresh event

## Frontend

- Design: editorial newspaper aesthetic with Playfair Display (serif headlines) and DM Sans (everything else)
- Accent: bright cyan (#00E5FF), newsletter orange (#E8813B), podcast green (#3ECF6E)
- Layout: masonry grid for article cards, collapsible icon-only sidebar, solid top bar
- Feed summary: two-segment (NEWS SUMMARY + INFORMATIVE HIGHLIGHTS), collapsible, auto-generated by pipeline
- Article search: backend q parameter on GET /api/articles, debounced frontend input
- Add source: Taddy search + manual URL validation, 3-step modal flow

## Prohibitions

- NEVER use local Whisper — it has been removed from the project
- NEVER modify DB schema outside db.py _run_migrations()
- NEVER hardcode API keys — read from settings/environment
- NEVER use datetime.utcnow() — it is deprecated
- Do not attempt programmatic login to Colossus — Cloudflare blocks it. Use RSS fallback only.
- Do not attempt Taddy transcript fetches — they return 400 for most shows
- Do not create files in project root — utilities in scripts/, config in src/config/
- Do not add frontend dependencies without checking bundle size impact
- Do not overwrite user-configured transcript_priority, active, or content_category in seed_from_yaml
